Scripted Installation

The scripts in this folder can be used to install JBoss Tools via commandline into a new or existing Eclipse 4.3 installation.

These files are simply wrappers which use Eclipse to call Ant, passing in variables for source, target, and what to install (if not everything). Run the script as `ant -f install*.xml help` for example usage.

  • install.cmd (Windows)
  • install.sh (Mac or Linux)
  • install.xml (Ant script to install JBDS from archive)
  • director.xml (Ant script to install from any update site)

Usage examples


  • Install all of JBoss Tools into Eclipse Standard bundle 
    target=${HOME}/eclipse/eclipse43; cd ${target}; rm -fr ${target}/eclipse
tar xzf /tmp/Eclipse_Bundles/eclipse-standard-kepler-R-linux-gtk-x86_64.tar.gz
${target}/eclipse/eclipse -consolelog -nosplash -data /tmp -application org.eclipse.ant.core.antRunner -f \
  install.xml -DsourceZip=/tmp/jbosstools-Update-4.1.0.Final_2013-07-19_19-47-52-B380.zip \
  -DotherRepos=http://download.jboss.org/jbosstools/updates/stable/kepler/central/core/ \
  -DtargetDir=${target}/eclipse/

  • Install JBoss Tools Central and Subclipse w/ SVNKit adapter into Eclipse JEE bundle 
    target=${HOME}/eclipse/eclipse43; cd ${target}; rm -fr ${target}/eclipse
tar xzf /tmp/Eclipse_Bundles/eclipse-jee-kepler-R-linux-gtk-x86_64.tar.gz
${target}/eclipse/eclipse -consolelog -nosplash -data /tmp -application org.eclipse.ant.core.antRunner -f \
  install.xml -DsourceZip=/tmp/jbosstools-Update-4.1.0.Final_2013-07-19_19-47-52-B380.zip \

Variations


For a simpler Ant script which can be used w/ any URL instead of a previously downloaded update site archive as input, see director.xml. Note that director.xml does not throw helpful error messages if commandline params are invalid or missing.
